# ♟️ Moltbot Investment System - Requirements & Backlog

## 1. 研发优先级矩阵 (P-Level Matrix)
* **P0 (Fatal):** 系统崩溃、核心数据流断裂（需立即修复）。
* **P1 (Critical):** 核心逻辑失效/严重误导（如防守资产组全部被错误熔断）。
* **P2 (Major):** 重要功能升级、算法深化、高 ROI 新特征、另类数据接入。
* **P3 (Minor):** 常规逻辑微调、UI/UX 优化。
* **P4 (Trivial):** 技术债清理、代码重构、远期构想。

## 2. 需求池 (Backlog)

### 🟡 [P2 - Major] 架构级: 叙事强度因子 (Narrative Momentum) 深度开发
* **关联模块:** `my_stock_pool.py` / `2_我的自选股池.py` / `3_个股深度猎杀.py`
* **问题描述:** 现有的"叙事热度 (10分)"为静态赋分，缺乏与实际资金动量的动态咬合。将来需用于 C/D 组的动量判定。
* **工程动作:**
  1. 设计叙事因子的动态衰减与爆发算法（基于短周期成交量异动 Vol_Z）。
  2. 将叙事逻辑判定下放至 Page 3 (个股深度猎杀)，使其与 POC 筹码结构发生化学反应，形成"有叙事支撑的突破"高权重评分。

### 🟡 [P2 - Major] 另类数据源: 知名推特博主 Alpha 信号爬虫与降噪
* **关联模块:** 独立数据管道 (`plugins/twitter_crawler.py`) / `core_engine.py`
* **问题描述:** 主理人需要实时获取 Crypto/宏观大 V 的前瞻性投资观点，但信息流中充斥大量无价值的吹水与广告，人工筛选信噪比极低。
* **工程动作:** 1. 构建爬虫管道：定向监听白名单博主推文列表。
  2. LLM 信号提纯网关：引入大语言模型（如 Claude 3）对推文进行机器阅读，要求其仅提取"标的"、"看多/看空方向"、"核心逻辑"，强制过滤生活分享与营销内容。
  3. 资产映射：将提取的有效机会结构化，作为独立的事件驱动因子，推送至前端雷达。



### 🔵 [P3 - Minor] 漏斗筛选逻辑颗粒度细化
* **关联模块:** 核心量化引擎
* **问题描述:** 当前系统的筛选逻辑（如 30-40-20-10 霸权模型）在特定宏观环境下可能不够细腻。
* **工程动作:** 梳理各梯队资产（A-D组）的过滤阻力，根据主理人反馈调整入场线/淘汰线的阈值，提升系统信噪比。


  
### ⚪ [P4 - Trivial] 架构级: 引入 VectorBT 实现动态滚动回测 (Dynamic Backtest)
* **关联模块:** `app.py` / `core_engine.py`
* **问题描述:** 当前策略净值回测逻辑为"时点回溯投射 (静态回测)"，存在生存者偏差，无法真实模拟带时间轴的滚动调仓。
* **工程动作:** 1. 引入 `vectorbt` 向量化回测框架，替代现有的 Pandas 静态回测公式。
  2. 实现毫秒级的真实事件驱动/滚动调仓净值回测，彻底消除未来函数，为后续策略实盘对接与机构募资提供硬核支撑。

## 3. 架构演进路线图 (Architecture Roadmap)

### 🚨 第二阶段：Alpha 引擎的阿喀琉斯之踵（为什么未来必须换数据源？）

Page 4（同类资产竞技场）为 C 组（时代之王）设计的终极指标是 **Forward EPS（预期每股收益）增速**，这正是 yfinance 这根免费水管的致命死穴。

yfinance 本质上是爬虫，从雅虎财经网页硬刮数据。刮几只股票的收盘价没问题，但若每天刮美股几千只股票"未来 12 个月华尔街分析师一致预期 EPS"这种深度基本面数据，雅虎财经会直接封 IP（Rate Limit）或返回大量空值（NaN）。底层 Forward EPS 取不到数，ScorecardC 瞬间瘫痪。

#### 🔌 OpenBB：不是水源，是"超级路由器"

OpenBB 本身不生产数据，它是一个**标准化数据聚合终端**。专业资管领域要获取最准的 EPS 预期、最快的财报数据，需购买专业 API（如 Financial Modeling Prep (FMP)、Polygon.io、Alpha Vantage 等，哪怕几十美元/月的平民级机构接口）。

* **没有 OpenBB：** 需要自己学 FMP、Polygon 等各家接口，代码变成屎山。
* **有了 OpenBB：** 在 OpenBB 中填入各 API 密钥，之后只需调用 `obb.equity.fundamental.estimates(symbol="NVDA")`，OpenBB 自动从 FMP 拉回精准 Forward EPS，并自动清洗成干净 Pandas 表格喂给系统。

#### 📍 首席架构师实战路线图："先上车，后换引擎"

**现在（Mock / yfinance 阶段）：**
继续用 yfinance 搞定量价，FRED 搞定宏观。C 组的 EPS 数据暂时用静态表格或简单缓存替代（或用 `yfinance.Ticker().info` 刮一点算一点），**先让整个 UI 交互、数据流转、自动旁白生成、饼图分配的逻辑层全部写死并跑通**。

**未来（OpenBB 赋能阶段）：**
当 Moltbot 逻辑完美无瑕、准备实盘或服务高净值客户时，再做一次**数据源重构（Data Source Refactoring）**——只需将底层取数函数从 yfinance 替换为 OpenBB，逻辑层完全不动。
