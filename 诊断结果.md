诊断结果：期望与实现之间的结构性错配
你期望的持有周期是：

级别	期望持有周期	评审节奏
A 压舱石	~2 年	季度
B 大猩猩	~1 年	月/季
C 时代之王	~半年	月度
D 预备队	~几个月	周度
但你实际使用的判定信号是周度甚至日度频率的短周期指标，而且没有任何"防抖"机制。这就像用秒针来判断应该几点出门——它每秒都在动，你永远也做不了决定。

问题在哪？

B 级的 curr > MA200：价格在 MA200 附近震荡时，一个月站上、下个月跌破，资产在 B 和 C 之间反复跳。没有滞后带 (hysteresis)。
A 级的 is_bullish (MA20 > MA60)：这是一个月度级别都会翻转的短周期信号，用它来守护一个期望持有 2 年的级别，频率完全不匹配。
C 级的 full_uptrend (MA20 > MA60 > MA250)：三条均线的多头排列非常脆弱，MA20 哪怕短暂跌破 MA60 一天就丢失 C 级资格。
根源 2：级联漏斗的放大效应
A→B→C→D 的级联结构意味着：一个资产从 B 级掉出后，不是"降级到 B-"，而是进入一个完全不同评分体系的 C 级或 D 级。

比如 NVDA：某月 curr > MA200 通过 B 级，下月跌破 MA200 后 RS 排名足够高就进 C 级。但 B 组用的是「护城河质量指数」（股息率+回撤+夏普），C 组用的是「慢变量霸权指数」（EPS增速+RS120+宏观匹配），两个完全不同的评价体系。你看到的"BC 乱成一锅粥"就是这个原因。

def _minmax_norm(series: pd.Series) -> pd.Series:
    lo, hi = series.min(), series.max()
    if hi == lo:
        return pd.Series([50.0] * len(series), index=series.index)
    return (series - lo) / (hi - lo) * 100.0
这意味着即使一只资产自身指标不变，只要同赛道有一个新成员进来或旧成员退出（因为根源 1/2 导致的跳级），所有人的分数都会重新洗牌。粗筛的不稳定被这个机制进一步放大。

根源 5：宏观二元匹配 (20%) + 数据缓存不一致
C 组的宏观匹配是 100 或 0 的二元判定，剧本一变所有标的的 20% 评分瞬间翻转。同时 get_stock_metadata TTL=24h 而 get_arena_*_factors TTL=4h，同一资产的股息率在粗筛和细筛中可能基于不同时点数据。

我建议你分三步走
第一步（投入产出比最高）：给粗筛加"滞后带"
原理很简单：进入一个级别需要通过严格阈值，但退出需要跌破更宽松的阈值。在 classify_asset() 中增加一个 prev_cls 参数：

指标	进入阈值	退出阈值	效果
B: curr vs MA200	> MA200	< MA200 × 0.95	跌破 5% 才降级
A: 最大回撤	< 15%	> 20%	5% 缓冲带
A: SPY 相关性	< 0.65	> 0.75	0.10 缓冲带
C: RS 排名	≤ 20%	> 35%	15% 缓冲带
这一步不改变任何评分公式，只需要在 classify_asset 和 session_state 中保存上一期分类结果。

第二步：用慢指标替换快指标
A 级的 is_bullish (MA20>MA60) → 改为 MA60 > MA200 或直接只认股息率（你想持有 2 年的东西不应该依赖月度均线）
C 级的 full_uptrend (MA20>MA60>MA250) → 简化为 curr > MA250（单一年线，更稳定）
B 级增加市值的滞后缓冲（进入 >1000 亿，退出 <800 亿）
第三步：细筛改用固定锚点归一化
把 _minmax_norm 从纯同期相对排名改为基于历史分位数的绝对锚点。比如用过去 2 年全池数据确定各因子的 5th/95th 百分位作为归一化的 lo/hi，这样即使参赛者变化，单个资产的分数基本稳定。

同时建议 C 组 Forward EPS 权重从 50% 降到 25-30%，腾出来的权重给 RS_250d（年度超额收益）这种更慢的因子。

总结一句话：你的系统不是评分逻辑有问题，而是「信号频率」和「期望持有周期」严重不匹配。第一步加滞后带是成本最低、见效最快的改动，改完之后再逐步替换快指标和归一化方式。
