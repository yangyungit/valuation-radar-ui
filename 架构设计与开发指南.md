叙事与舆情共振引擎 - 架构设计与开发指南
一、 项目定位与核心目标
本项目为 Moltbot 个人量化投研系统的新增核心模块。

物理落位：前端新增 pages/7_叙事与舆情雷达.py；后端 valuation-radar 新增 NLP 数据流水线与 SQLite/DuckDB 持久化层。

核心目标：重构 D 组（预备队）的入选逻辑。摒弃“纯量价动量”的盲目扫单，升级为**“高价格动量 + 高舆情爆发”的双因子共振交叉验证**，旨在精准捕获具有强宏观/产业逻辑支撑的“大象起舞”级别交易机会。

二、 核心设计哲学 (必须严格遵守)
1. 词与标的的“政教分离”原则 (Separation of Church and State)
绝对禁止硬编码绑定：系统初始的叙事字典中，只有词（概念），没有标的（股票代码）。

动态相爱（动态搭桥）：词（宏观/产业叙事）与标的（Ticker）的关联，必须通过每日新闻抓取中的**“NLP 文本共现 (Co-occurrence)”**来动态建立。随着市场炒作逻辑的漂移，系统能自动识别出新龙头并抛弃旧龙头。

2. 白盒化第一基本法 (White-box Protocol)
拒绝任何不可解释的黑盒聚类算法（如 K-Means 自动切分板块）。

任何导致资产晋级 D 组的底层动因（命中哪个词、Z-Score 多少、情感极性如何、共振相关系数多少），必须在 UI 端以清晰的卡片形式呈现（“去裸奔化”）。

3. 半人马工作流 (Human-in-the-loop)
AI 起草，人类审批：系统通过无监督学习（N-gram）或 LLM 发现异动生词或提议新概念簇，但必须经过 CIO（用户）在前端界面的点击**[批准]**，才能写入核心字典生效。

系统吹哨，人类主分：系统每日计算簇内资产的“走势撕裂度”，若发现同概念下资产走势分化，系统只负责弹窗预警，由 CIO 人工决定是否向下细分颗粒度。

三、 概念定义与系统字典架构
1. 映射簇的颗粒度与树状结构 (Taxonomy)
叙事字典必须采用三层嵌套的 JSON 结构，自顶向下进行逻辑归因：

L1 宏观母题 (Theme)：如 #THEME_AI (AI革命), #THEME_MACRO (宏观周期)

L2 产业干线 (Sector)：如 #AI_Compute (算力硬件), #AI_Power (能源基建)

L3 催化剂/特征词 (Keywords)：这是最小计算单元。如 #AI_Power 下属的 "Nuclear", "Grid", "Data Center Heat"。

2. 核心量化指标定义
Attention_Z_Score (注意力爆发度)：某特征词簇今日的新闻提及量，相对于过去 30 天均值和标准差的 Z-Score。（阈值参考：> 2.0 视为显著异动）。

Sentiment_Score (情绪极性)：通过 FinBERT 或同类轻量级金融 NLP 模型计算出的文本情感得分，范围 [-1.0, 1.0]。

Resonance_Score (共振系数)：过去 20 个交易日内，某股票的日收益率走势与某叙事簇 Attention_Z_Score 走势的皮尔逊相关系数（Pearson Correlation）。

四、 核心工作流：从文本到交易信号
请 Cursor 按照以下流水线进行后端架构设计：

数据采集 (Data Sourcing - 异步执行)：

每日盘后或按需触发，拉取 my_stock_pool.py 中所有资产相关的财经新闻标题及摘要（Yahoo Finance/Finnhub）。

提纯与匹配 (NLP Processing)：

已知词匹配：用预设的“L3 催化剂字典”清洗新闻，计算各叙事簇的 Attention_Z_Score 和情感。

未知词发现：统计字典外的高频 N-gram 词组，抛入“待审批收件箱”。

建立共现矩阵：扫描同一篇新闻中同时出现的 Ticker 和 特征词，建立当日的关联映射。

终极验证与 D 组晋级 (Execution)：

触发条件：Ticker 价格动量达标 AND 所绑定的叙事簇 Attention_Z_Score > 2.0 AND 两者相关系数 (Resonance_Score) > 0.6。

满足条件的资产，正式晋级至 Page 2 的 D 组，并生成白盒化归因报告。

五、 系统前后端架构设计约束
后端 (valuation-radar - FastAPI)：

引入持久化层：由于计算 Z-Score 和保存“待审批字典”需要历史状态，必须引入 SQLite 或 DuckDB，严禁继续使用内存 {} 作为历史状态容器。

异步非阻塞：爬虫和 NLP 模型调用耗时极长，必须使用 BackgroundTasks 或定时任务在后台静默写入数据库。API 接口只负责瞬间读取计算好的最新结果。

前端 (Page 7 叙事与舆情雷达 - Streamlit)：

视图 1：全局叙事热力图：展示当前市场上最热的 L1/L2 概念簇及其 Z-Score 走势。

视图 2：大象雷达 (异动标的)：展示今日发生“高动量+高舆情”共振的标的。

视图 3：CIO 审批控制台：一个 CRUD 表格，展示系统抓取到的异动生词或内部走势撕裂预警，供用户点击“批准入库”或“忽略”。

六、 给 Cursor 的特别开发嘱咐
最小化侵入：不要大面积修改现有的 core_engine.py 中已经稳定运行的 VCP 识别和基础的 DataFrame 数据获取逻辑。新逻辑应作为外挂模块（如 narrative_engine.py）接入。

容错处理：外部 NLP API（或新闻抓取 API）极不稳定，所有请求必须包裹在严格的 try-except 中。如果抓取失败，系统应降级使用纯价格动量逻辑，决不能导致前端页面白屏。

UI 渲染规范：严格遵守原有项目的审美和代码规范（物理文件名不带 Emoji，深色主题，使用原生的 HTML/CSS 进行卡片渲染，如 <div class='detail-box'>）。
