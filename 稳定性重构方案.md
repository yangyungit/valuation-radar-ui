# 估值雷达 — 分拣与细筛稳定性重构方案

> **目标**：让 ABCD 分级结果的变化频率与期望持有周期匹配：  
> A 级（压舱石）稳定 ~2 年，B 级（大猩猩）稳定 ~1 年，C 级（时代之王）稳定 ~半年，D 级（预备队）稳定 ~几个月。
>
> **核心原则**：筛选逻辑（"这个标的属于哪个级别"）使用慢指标；交易逻辑（"什么时候上/下车"）使用快指标。二者分离。

---

## 一、架构改动：级联漏斗 → 并行独立评估

### 1.1 当前架构问题

当前 `classify_asset()` 是 A→B→C→D 级联漏斗，通过某级即停止。后果：
- NVDA 某月在 MA200 之上 → B 级（按护城河质量评分），下月跌破 MA200 → C 级（按 EPS 增速评分），评分体系完全不同，下游 Page 4 结果剧烈震荡。
- 资产在相邻级别间来回跳，每次跳级都改变两个赛道的参赛者名单，通过 MinMax 归一化波及所有人的分数。

### 1.2 新架构：并行独立评估

**每只资产独立评估全部 4 套标准**，可以同时符合多个级别。

```
旧：A? → 不通过 → B? → 不通过 → C? → 不通过 → D? → 不通过 → "?"
新：同时评估 A✓/✗、B✓/✗、C✓/✗、D✓/✗ → 得到 qualifying_grades = ["B", "C"] 等
```

数据结构变更：

```python
# 旧：每个资产只有一个 cls
all_assets[ticker] = {"cls": "B", ...}

# 新：每个资产有 qualifying_grades 列表 + primary_cls（最高优先级）
all_assets[ticker] = {
    "qualifying_grades": ["B", "C"],  # 所有通过的级别
    "primary_cls": "B",               # 显示用主级别（优先级 A>B>C>D）
    ...
}
```

**Page 2 展示**：主级别高亮 + 副级别徽章。同一资产可以出现在多个分组中。  
**Page 4 竞技场**：每个赛道包含所有 qualifying 该级别的资产（允许重复出现在多个赛道）。

## 二、粗筛指标改造：慢指标 + 滞后带

### 2.1 慢指标替换

| 级别 | 当前指标（太快） | 改为（更慢） | 理由 |
|------|-----------------|-------------|------|
| A: 收益来源 | `div_yield >= 1% OR MA20 > MA60` | `div_yield >= 1% OR MA60 > MA200` | MA60>MA200 是季度级信号，匹配 A 级 ~2 年持有 |
| B: 长线趋势 | `curr > MA200` | `curr > MA200`（不变，靠滞后带稳定） | MA200 本身已是慢指标 |
| C: 主升浪 | `MA20 > MA60 > MA250`（三线多头） | `curr > MA250`（站上年线即可） | 年线更稳定，MA20/MA60 交叉是月度噪音 |
| D: 短期动量 | 不变 | 不变 | D 级本就是周度评审 |

**对应 `compute_metrics()` 修改**：增加 `slow_bullish` 字段 = `MA60 > MA200`。

### 2.2 滞后带（Hysteresis）

需要在 `classify_asset()` 中增加 `prev_grades: list` 参数，对每个级别分别判定：

| 级别 | 条件 | 进入阈值（严格） | 退出阈值（宽松） | 缓冲效果 |
|------|------|-----------------|-----------------|---------|
| A | 最大回撤(1Y) | < 15% | > 20% | 5% 缓冲 |
| A | SPY 相关性 | < 0.65 | > 0.75 | 0.10 缓冲 |
| A | 收益来源 | div≥1% 或 MA60>MA200 | div<0.5% 且 MA60<MA200 | 双降才退出 |
| B | 市值 | > $1000亿 | < $800亿 | $200亿 缓冲 |
| B | 3Y 最大回撤 | < 40% | > 50% | 10% 缓冲 |
| B | 长线趋势 | curr > MA200 | curr < MA200 × 0.95 | 5% 缓冲 |
| C | RS 排名 | ≤ 20% | > 35% | 15% 缓冲 |
| C | 年线支撑 | curr > MA250 | curr < MA250 × 0.95 | 5% 缓冲 |
| D | 动量/波动率 | 不变 | 不变 | D 级无需滞后 |

**实现要点**：
- `classify_asset_parallel(m, div_yield, mcap, prev_grades=None)` → 返回 `(qualifying_grades, details_dict)`
- 当 `prev_grades` 中已包含某级别时，该级别使用退出阈值；否则使用进入阈值
- `prev_grades` 持久化到 `data/prev_classification.json`，每次运行时读取上一期结果

### 2.3 `screener_engine.py` 改造伪代码

```python
def classify_asset_parallel(
    m: dict, div_yield: float, mcap: float,
    prev_grades: list = None,
) -> tuple[list, dict]:
    """并行独立评估 ABCD 四个级别，返回 (qualifying_grades, all_details)。"""
    if not m.get("has_data"):
        return [], {"error": "数据不足"}
    
    prev = set(prev_grades or [])
    grades = []
    all_details = {}
    
    # ── A 级独立评估 ──
    was_a = "A" in prev
    a_income_enter = div_yield >= 1.0 or m.get("slow_bullish", False)  # MA60 > MA200
    a_income_exit  = div_yield < 0.5 and not m.get("slow_bullish", False)
    a_dd_enter     = m["max_dd"] < 15.0
    a_dd_exit      = m["max_dd"] > 20.0
    a_corr_enter   = m["spy_corr"] < 0.65
    a_corr_exit    = m["spy_corr"] > 0.75
    
    if was_a:
        a_pass = not a_income_exit and not a_dd_exit and not a_corr_exit
    else:
        a_pass = a_income_enter and a_dd_enter and a_corr_enter
    
    all_details["A"] = {...}  # 各关卡详情
    if a_pass:
        grades.append("A")
    
    # ── B 级独立评估 ──
    was_b = "B" in prev
    b_mcap_enter = mcap > 1e11
    b_mcap_exit  = mcap < 8e10
    b_dd3y_enter = m.get("max_dd_3y", 99) < 40.0
    b_dd3y_exit  = m.get("max_dd_3y", 99) > 50.0
    b_ma200_enter = m.get("ma200") is not None and m["curr"] > m["ma200"]
    b_ma200_exit  = m.get("ma200") is not None and m["curr"] < m["ma200"] * 0.95
    
    if was_b:
        b_pass = not b_mcap_exit and not b_dd3y_exit and not b_ma200_exit
    else:
        b_pass = b_mcap_enter and b_dd3y_enter and b_ma200_enter
    
    all_details["B"] = {...}
    if b_pass:
        grades.append("B")
    
    # ── C 级独立评估 ──
    was_c = "C" in prev
    c_rs_enter    = m["rs_rank_pct"] <= 0.20
    c_rs_exit     = m["rs_rank_pct"] > 0.35
    c_ma250_enter = m.get("ma250") is not None and m["curr"] > m["ma250"]
    c_ma250_exit  = m.get("ma250") is not None and m["curr"] < m["ma250"] * 0.95
    
    if was_c:
        c_pass = not c_rs_exit and not c_ma250_exit
    else:
        c_pass = c_rs_enter and c_ma250_enter
    
    all_details["C"] = {...}
    if c_pass:
        grades.append("C")
    
    # ── D 级独立评估（无滞后带）──
    d_mom = m["mom20"] > 8.0 or m["mom5"] > 5.0
    d_hv  = m.get("hv_60d", 0) > 0.25
    
    all_details["D"] = {...}
    if d_mom and d_hv:
        grades.append("D")
    
    return grades, all_details
```

## 三、细筛评分改造

### 3.1 锚点归一化替代纯 MinMax

**当前问题**：`_minmax_norm` 的 lo/hi 完全取决于本期参赛者，人员进出就全洗牌。

**改造方案**：使用历史分位数作为固定锚点。

```python
# 锚点配置：每个因子的 [low_anchor, high_anchor]
# 基于全池历史 2 年数据的 5th / 95th 百分位确定
FACTOR_ANCHORS = {
    # A 组
    "max_dd_inv":   (-0.25, -0.03),   # 最大回撤（越接近 0 越好）
    "div_yield":    (0.0, 5.0),        # 股息率 %
    "spy_corr_inv": (-0.8, 0.3),       # SPY 相关性（取反，越低越好）
    "vol_inv":      (0.40, 0.08),      # 年化波动率（取反，越低越好）
    # B 组
    "sharpe_1y":    (-0.5, 2.5),
    "log_mcap":     (9.0, 12.5),
    "eps_stability": (0.5, 10.0),
    # C 组
    "eps_growth":    (-20, 40),         # EPS 增速 %
    "rs_120d":       (-15, 30),         # 120 日超额收益 %
    "rs_250d":       (-20, 40),         # 250 日超额收益 %（新增因子）
    # D 组
    "vol_z":         (-1.0, 4.0),
    "rs_20d":        (-10, 25),
    "ma60_breakout": (0, 100),          # 已经是 0-100 打分
}

def _anchor_norm(series: pd.Series, lo: float, hi: float) -> pd.Series:
    """锚点归一化：用固定的历史分位数作为 lo/hi，而非同期 min/max。
    超出锚点范围的值会被 clip 到 [0, 100]。"""
    if abs(hi - lo) < 1e-9:
        return pd.Series(50.0, index=series.index)
    normed = (series - lo) / (hi - lo) * 100.0
    return normed.clip(0, 100)
```

**过渡方案**：初始锚点值可以先用合理的经验值硬编码（如上表），后续加一个"校准"功能，用回填的历史数据自动计算 5th/95th 百分位并更新锚点。

### 3.2 C 组评分卡重构（降低 EPS 权重，增加 RS_250d）

| 因子 | 当前权重 | 新权重 | 变化说明 |
|------|---------|-------|---------|
| Forward EPS 增速 | 50% | **25%** | 减半，降低对华尔街一致预期的依赖 |
| log₁₀(MCap) | 15% | **15%** | 不变 |
| RS₁₂₀ 中长线强度 | 15% | → 替换为 **RS₂₅₀ 年度超额收益 20%** | 更慢的动量因子 |
| 宏观顺风 | 20% | **20%** | 不变 |
| **新增：RS₂₅₀ 年度超额收益** | — | **20%** | 250 日 vs SPY 超额收益，半年才显著变化 |

新公式：
```
Score_C = 25% × Z_ForwardEPS + 15% × log₁₀(MCap)
        + 20% × RS₂₅₀ + 20% × Fit_Macro + 20% × RS₁₂₀  -- 备选方案
```

> 备选方案（如果 RS₁₂₀ 和 RS₂₅₀ 共线性太高）：用 RS₂₅₀ 替换 RS₁₂₀，而非两者并存。
> 那就变成：25% EPS + 15% MCap + 20% RS₂₅₀ + 20% 宏观 + 20% 盈利质量（如 ROE/ROIC）

### 3.3 其余赛道评分卡（A / B / D）

A 组和 B 组当前评分逻辑基本合理，主要受益于锚点归一化的改造（根源 3 的修复）。
D 组天然是短周期，不需要额外稳定化。

## 六、设计要点备忘

### 6.1 筛选逻辑 vs 交易逻辑（分离）
- **筛选逻辑**（本方案改造的部分）：回答"这个标的属于什么级别"，使用慢指标（MA60/MA200/MA250、年度 RS、年化回撤等），评审频率 = 月度或季度。
- **交易逻辑**（不在本方案范围内，但预留接口）：回答"什么时候上/下车"，使用快指标（周线 MA20 下穿 MA60 触发卖出等），评审频率 = 周度或日度。
- 两者不应混用。Page 2/4 只做筛选，交易信号留给 Page 5（个股深度猎杀）或未来的交易信号模块。

### 6.2 并行分类下的"主级别"
- `primary_cls` = qualifying_grades 中优先级最高的（A > B > C > D）
- 用于 Page 2 的主显示色和分组默认展示
- 但用户可以在任何分组中看到该资产（如果它 qualify 了该级别）

### 6.3 滞后带在历史回填中的处理
- 历史回填按月份正序执行（从最早月开始），每个月的分类结果作为下个月的 `prev_grades`
- 第一个月没有 prev_grades，全部使用进入阈值（等同于冷启动）

### 6.4 锚点归一化的初始值
- 初始硬编码合理经验值即可
- 后续加一个校准按钮：用全量回填数据计算各因子的 5th/95th 百分位，写入 `data/factor_anchors.json`
- `_anchor_norm()` 优先读文件锚点，文件不存在时 fallback 到硬编码默认值
