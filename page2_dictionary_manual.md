# Page 2 舆情监控词典说明书

## 1. 文档目的

这份说明书用于完整解释本项目 `Page 2 - 舆情监控` 中“词典”模块的业务定位、数据结构、工作流程、关键公式、管理动作、质量要求，以及它为什么不是一个普通的关键词表，而是整个叙事引擎和选股链路的核心基础设施。

这份文档还有第二个用途：

1. 作为团队内部对词典机制的统一认知底稿。
2. 作为后续与 ChatGPT 深化词典质量时的任务说明书。
3. 作为“为什么必须持续打磨词典，而不是只调 UI 或只调分数阈值”的论证材料。

## 2. 一句话定义

`Page 2` 的词典，本质上是一套 **L2 板块 - L3 关键词** 的动态叙事知识库。

它承担四个角色：

1. 它是新闻文本进入系统后的第一层结构化入口。
2. 它是新词发现、旧词追踪、主题归类、热度计算的共同基座。
3. 它是 Page 2 叙事雷达和 Page 3 D 组共振猎场之间的数据桥梁。
4. 它最终会影响 C/D 两组标的的选择逻辑，尤其是 D 组的共振排序和 C/D 卫星池的组合结果。

## 3. 为什么词典极其重要

很多系统把词典理解为“给新闻打标签的辅助表”。这个项目里不是这样。

在本项目中，词典的重要性体现在以下几个层面：

### 3.1 它决定系统有没有“主题理解能力”

没有词典，系统只能看到新闻很多、价格很强，但不知道这些动静到底属于：

- AI 电力
- 半导体周期
- 地缘政治
- 金融信用扩张
- 核能基础设施

词典把原始新闻流从“杂乱文本”变成“可归因的市场叙事”。

### 3.2 它决定系统能不能把“词”变成“板块”

系统不是只关心某个词热不热，而是关心：

- 某个 L2 板块最近是不是在升温
- 这个板块的热度是集中爆发还是分布共振
- 这个板块下面最活跃的 L3 词是什么
- 某只股票究竟更像哪个 L2 的受益者

也就是说，词典不是孤立词汇表，而是从 L3 汇总出 L2 因子的上游结构。

### 3.3 它决定舆情因子的信噪比

词典质量高，得到的是：

- 高相关关键词
- 清晰的主题边界
- 低噪音命中
- 更稳定的 L2 热力分
- 更可信的 D 组共振分

词典质量低，得到的是：

- 泛词、歧义词、人名地名混入
- L2 归属混乱
- 无关新闻被错算为热点
- 真正主线被稀释
- D 组标的被错配到错误板块

### 3.4 它不是展示层配置，而是交易前置基础设施

Page 2 会把 L2 热力排行写入 `session_state["narrative_heat_ranking"]`，Page 3 再据此做 D 组共振计算。这意味着词典并不止影响页面展示，而是已经影响到选股排序和最终组合解释。

## 4. 当前 Page 2 词典在系统中的位置

从代码和页面结构看，Page 2 的核心流程是：

1. 新闻采集
2. 已知词匹配
3. TF-IDF 新词发现
4. LLM 质检
5. 新主题提案 / 待归类中心
6. L2/L3 词典管理
7. 旧词统计
8. L2 叙事雷达与历史主线雷达

也就是说，词典同时处在：

- 输入侧：承接新闻文本
- 中间侧：组织新词、旧词和主题归类
- 输出侧：生成 L2 热力、叙事排行、共振输入

## 5. 词典的结构设计

### 5.1 基本层级

项目设计上采用三层 taxonomy：

1. `L1`：宏观母题
2. `L2`：产业干线 / 赛道主线
3. `L3`：催化剂、表征词、叙事触发词

但在当前前端落地里，最核心、最常用的是 `L2 + L3`。

### 5.2 L2 的业务含义

L2 不是普通分类标签，而是“可被计算热度和动量的叙事板块单元”。

例如一个 L2 可以代表：

- `Nuclear_&_AI_Power`
- `Semi_Memory_&_Cycles`
- `Macro_Liquidity`
- `Trade_&_Geopolitics`

L2 是 Page 2 雷达、Page 3 共振、后续叙事选股的真正承载层。

### 5.3 L3 的业务含义

L3 是某个 L2 下面最小的可命中词条，通常包括：

- 技术词
- 设施词
- 产品词
- 政策词
- 催化事件词
- 产业链节点词
- 市场常用缩写

L3 不是越多越好，而是要保证“命中后真的能代表该 L2 的叙事活动”。

## 6. 词典中的来源与状态

当前词典并不是单一来源人工维护，而是一个动态增量系统。

### 6.1 词条来源

当前前端明确支持以下来源：

1. `seed`
   初始种子词。
2. `approved`
   人工审批通过的词。
3. `bottom_up`
   由孤儿词/自底向上提案进入的词。
4. `tfidf_auto`
   TF-IDF 发现后通过质检自动晋升的词。

这四类来源很重要，因为它们代表词条“可信度、成熟度、发现路径”的不同。

### 6.2 词条状态

当前管理界面支持以下状态：

1. `active`
   活跃词条，参与匹配与统计。
2. `dormant`
   休眠词条，可恢复。
3. `archived`
   归档词条，可恢复。
4. `seed_only`
   仅种子蓝图，尚未真正激活。

### 6.3 噪音黑名单

词典管理中不仅可以删除词条，还可以“删除并标噪”。一旦进入噪音黑名单，后续系统应避免再次把这些词视为有效候选。

这意味着词典的维护不仅是“加词”，也是“持续去噪”。

## 7. Page 2 词典的完整工作流

### 7.1 新闻采集层

Page 2 白盒说明里明确写到，系统会从多源抓取财经新闻，包括：

- GDELT GKG
- GDELT DOC API
- Google RSS
- Finnhub
- Alpaca
- Polygon

词典本身不生成新闻，但它决定新闻进入系统后会被解释成什么。

### 7.2 已知词匹配

系统会对新闻标题和摘要做全文字符串扫描。

一旦命中词典，就会记录到 `keyword_match_log`，核心链路包括：

- 新闻 ID
- 关键词
- 来源
- 日期
- 情感分

这一步是“旧词统计”的数据基础。

所以旧词统计并不是静态报表，而是词典命中日志的白盒回放。

### 7.3 TF-IDF 新词发现

当某些词还不在词典里，但今天突然高频出现时，系统会把它们浮出来。

当前页面明确给出了定义：

- `TF = 今天含这个词的文章数 / 今日语料总篇数`
- `IDF = log((语料总数 + 1) / (含该词文章数 + 1)) + 1`
- `TF-IDF = TF × IDF`

此外还有一个非常关键的派生指标：

- `温度（burst_ratio） = 今日频率 / 历史均值`

其中：

- `温度 > 2.0` 通常视为放量信号
- `主题关联度 > 0.3` 代表它已经开始表现出主题归属可能性

这一步的意义不是直接拿新词做交易，而是为词典扩容提供候选池。

### 7.4 LLM 语义质检

系统当前会把 TF-IDF 候选词送入 LLM 质检层，目标包括：

1. 判断它是否和股票估值、产业事件、市场叙事有关。
2. 过滤泛词、功能词、人名、地名、娱乐词、无投资含义词。
3. 给出通过、拒绝、边界、待人工等裁决。

这一步非常关键，因为：

- 纯字符串高频，不等于有投资价值
- 高频词里混入噪音，会直接污染 L2 热度计算
- LLM 语义裁决是词典“扩容之前的防火墙”

### 7.5 新主题提案

如果系统发现一批词无法归入现有 L2，但它们彼此形成了较强集群，就会生成“新主题提案”。

审批通过后，可以：

1. 新建一个 L2 板块
2. 导入一组首批 L3 关键词
3. 让这个板块立即进入雷达和后续匹配逻辑

这一步很重要，因为市场主线是会演化的。词典必须具备“增量建库”的能力，而不是只有死板的固定 taxonomy。

### 7.6 待归类中心

当前系统还处理一类重要问题：词通过了质量门，但 L2 归属不明确。

这类情况包括：

1. `ambiguous_l2`
   两个 L2 竞争过近，系统无法自动裁决。
2. `borderline`
   统计层或语义层模糊，需要 CIO 处理。

这说明词典质量建设不是一次性动作，而是持续的人机共建流程。

### 7.7 词典全景管理

当前 Page 2 已支持完整的词典 CRUD：

1. 新建 L2 板块
2. 添加单个 L3
3. 批量添加 L3
4. 批量归档
5. 批量恢复
6. 批量迁移到其他 L2
7. 删除词条
8. 删除并标噪
9. 重命名 L2
10. 归档整个 L2
11. 彻底删除整个 L2

这说明系统已经把词典视为一套正式资产，而不是开发临时配置。

## 8. L2 层是如何被计算成“因子”的

很多人会误以为 L2 只是一个分类名。实际上，L2 在这个项目里已经是可量化因子。

当前 L2 维度至少承载以下关键数据：

1. `composite_heat`
   综合热力分。
2. `heat_momentum`
   热度动量，体现提及量环比变化率。
3. `mention_count`
   提及量。
4. `top_l3_keywords`
   当前最活跃的 L3 列表及其温度、文档数、连续天数。
5. `heat_type`
   集中式或分布式。
6. `attention_z_score`
   历史主线雷达中用于观察爆发程度的重要指标。
7. `quadrant`
   当前所处象限。
8. `dwell`
   在某一象限的连续停留时长。

这些都不是标签，而是能进入评分、排序、筛选和解释链路的因子化数据。

## 9. Page 2 如何把词典变成叙事热度

在 Page 2 的“叙事热度排行榜”里，当前核心公式是：

`叙事动量分 = (0.6 × composite_heat + 0.4 × momentum_boost) × 100`

其中：

- `composite_heat` 代表板块综合热度
- `momentum_boost` 只对正动量加速进行加成

这一步的关键意义在于：

1. 词典先组织出 L2/L3 结构
2. L3 的活跃情况汇总成 L2 的热力与动量
3. L2 再被排序成“叙事热度榜”
4. 这份榜单被下游页面消费

也就是说，词典不是静态库，而是 L2 因子生产引擎。

## 10. 词典如何影响 D 组标的选择

这是目前已经落地、而且必须讲清楚的部分。

在 Page 3 的 D 组“共振猎场”中，当前白盒公式是：

`共振分 = ScorecardD 动量分 × 叙事热度分 / 100`

而：

`叙事热度分 = L2 板块热力分 × L3 匹配度`

其中：

- `L2 板块热力分` 来自 Page 2 的 L2 叙事热度排序
- `L3 匹配度 = 共现命中 L3 词温度总和 / 该 L2 全部 L3 词温度总和`

这意味着：

1. 某只 D 组标的先有自己的价格动量分。
2. 系统再去查它近 7 天新闻共现了哪些关键词。
3. 用这些共现词去匹配各个 L2 下的 L3 词。
4. 选出叙事得分最高的 L2。
5. 再用该 L2 的热力分和匹配度，乘出该标的的叙事热度分。
6. 最终与 D 组动量分相乘，得到共振分。

结论非常明确：

**词典质量会直接改变 D 组标的的共振得分、匹配板块、排名先后和解释结果。**

### 10.1 词典质量差会怎样伤害 D 组

如果词典质量不好，会出现以下问题：

1. 该匹配的 L3 没收进去，导致匹配度偏低。
2. 不该匹配的泛词被收进去，导致错误命中。
3. 某个词被分错 L2，导致标的被挂到错误板块。
4. L2 下词条结构不完整，导致板块热度被低估。
5. L2 下噪音词太多，导致热度被虚高。

最终结果就是：

- 有动量有逻辑的票被漏掉
- 没逻辑的票被误推上来
- 归因解释不可信

## 11. 为什么要说 L2 数据因子会影响 C/D 两组标的选择

这一点需要分“当前已发生的影响”和“当前架构下已具备的联动影响”两层来讲。

### 11.1 当前已经直接发生的影响：D 组

这部分已经完全落地。

L2 的热力分、动量、Top L3 结构，会直接决定 D 组的：

- 匹配板块
- 叙事热度分
- 共振分
- 排名
- 白盒归因

### 11.2 当前已经存在的组合层联动影响：C/D 卫星池

在 Page 6 的仓位配置中，战术卫星池由：

- `C组 Top-2 = 30%`
- `D组 Top-2 = 20%`

共同组成。

因此，虽然当前代码里词典 L2 因子是显性注入 D 组，而不是显性注入 ScorecardC，但它已经通过以下方式影响 C/D 两组的最终标的选择结果：

1. D 组因为 L2 因子改变了入选顺序和最终 Top-2。
2. C/D 两组一起构成卫星池，D 组变化会改变整个卫星层的组合结构。
3. 组合层对“当前主攻是谁、备援是谁、资金如何分配”的认知会随之变化。

所以从组合视角说，L2 因子已经不只是影响 D 组一个榜单，而是在影响 C/D 卫星池最终呈现与配置结果。

### 11.3 下一步应该明确升级为：L2 因子前移到 C/D 共用筛选

如果你要和 ChatGPT 深化词典质量，我建议明确提出这个业务方向：

**词典 L2 数据因子不仅用于 D 组共振，还应逐步成为 C/D 两组共享的上游叙事门控因子。**

原因是：

1. C 组“时代之王”本质上应该代表宏观主线中的持续赢家。
2. 如果某个 C 组候选长期脱离主线 L2，可能只是价格强，不一定是“时代之王”。
3. 如果某个 L2 长期处于主线高热区，且某票同时具备 C 组慢变量优势，那么它更应被优先识别。

因此在业务表达上可以这样写：

**当前系统中，L2 因子已直接影响 D 组、并通过卫星池联动影响 C/D 两组的最终标的配置；下一阶段应把 L2 主线强度进一步前移，使其成为 C/D 两组共同消费的叙事上游因子。**

这句话既符合当前实现，又符合你要推动的升级方向。

## 12. 什么样的词典才算高质量

高质量词典不是“词越多越好”，而是满足下面六条：

### 12.1 边界清晰

每个 L2 都要有明确边界，避免：

- 一个词可以随意落到多个 L2
- 两个 L2 语义高度重叠
- 主题拆分过细或过粗

### 12.2 召回充分

同一个叙事在新闻里会有很多写法，词典必须覆盖：

- 正式术语
- 缩写
- 同义词
- 行业黑话
- 上游/下游表述
- 事件型表述

### 12.3 精度够高

不能为了召回，把大量泛词、背景词、语法词收进去。

例如：

- 太常见的功能词
- 很宽泛的经济词
- 纯公司名称但无主题表征
- 没有投资指向的人名地名

都容易污染匹配。

### 12.4 中英文与变体统一

财经新闻中经常混用：

- 全称 / 缩写
- 单复数
- 连字符 / 无连字符
- 大小写
- 英文 / 中文音译

如果不统一，会导致本来属于同一概念的热度被拆散。

### 12.5 具备时间韧性

一个好的词典不能只对某一轮短期炒作有效，而要尽量覆盖：

- 长周期主线词
- 中周期产业链词
- 短周期事件催化词

### 12.6 可解释

每个词最好都能回答三个问题：

1. 它为什么属于这个 L2？
2. 它命中后代表什么叙事动作？
3. 它容易和哪些 L2 混淆？

## 13. 与 ChatGPT 深化词典质量时，应该重点做什么

如果你的目标是让 ChatGPT 参与提升词典质量，建议把任务拆成以下几类，而不是只说“帮我补词”。

### 13.1 做 L2 边界校准

让 ChatGPT 判断每个 L2 是否：

- 边界过宽
- 边界过窄
- 与其他 L2 重叠
- 需要拆分
- 需要合并

### 13.2 做 L3 词条补全

让 ChatGPT 为每个 L2 补足：

- 核心主词
- 同义词
- 缩写
- 设备词
- 材料词
- 工艺词
- 监管词
- 财报催化词
- 订单/产能/投产/招标等事件词

### 13.3 做歧义词清洗

让 ChatGPT 专门识别：

- 跨行业通用词
- 容易误命中的词
- 太短的缩写词
- 需要上下文才成立的词

并建议：

- 保留
- 降权观察
- 改归属
- 删除
- 加入噪音库

### 13.4 做中英文映射统一

让 ChatGPT 统一输出：

- 英文标准词
- 常见新闻变体
- 中文显示名
- 可接受别名

### 13.5 做“叙事完整度”检查

让 ChatGPT 站在分析师角度判断：

“如果市场真正炒作这个 L2，新闻里最可能出现的词，我们现在是不是漏了？”

这类补全，往往比单纯看历史高频词更有价值。

## 14. 建议让 ChatGPT 输出什么格式

为了方便你回填到系统里，建议要求 ChatGPT 对每个 L2 输出以下结构：

```json
{
  "l2_name": "Nuclear_&_AI_Power",
  "l2_zh": "核能与AI电力",
```

## 15. 给 ChatGPT 的任务说明，建议你这样提

你可以把下面这段作为后续深化词典的任务底稿。

### 15.1 推荐任务口径

请基于“L2 板块 - L3 关键词”的叙事词典体系，帮助我提升词典质量。目标不是单纯扩词，而是同时提升边界清晰度、召回率、精确率和可解释性。请逐个 L2 输出：

1. 该 L2 的核心投资叙事定义
2. 必须保留的核心 L3
3. 应补充的同义词、缩写、事件词、政策词、产业链词
4. 容易误命中的歧义词
5. 建议移出或标噪的词
6. 是否需要拆分/合并该 L2
7. 对该 L2 会影响哪些下游因子做简要说明

并特别注意：

1. 词典质量不仅影响 Page 2 舆情监控，还会影响 L2 热力分、叙事动量分和 Page 3 D 组共振选股。
2. 从业务目标看，L2 数据因子应作为 C/D 两组共同消费的上游叙事因子，其中当前已直接作用于 D 组，并通过卫星池组合联动影响 C/D 两组最终标的选择。

## 16. 你在沟通里必须强调的核心观点

如果这份说明书是拿去跟别人对齐，最关键的不是介绍 UI，而是强调下面四句话：

1. 词典不是关键词表，而是叙事引擎的知识底座。
2. L3 决定命中精度，L2 决定因子聚合与板块热度。
3. 词典质量会直接改变 D 组共振选股结果。
4. L2 数据因子已经在组合层联动影响 C/D 两组，并应进一步升级为 C/D 共用的叙事上游因子。

## 17. 最终结论

如果把这个项目比作一套从新闻走向交易决策的机器，那么：

- 新闻源是原料
- LLM 是质检员
- 雷达是仪表盘
- 共振分是执行信号

而词典，就是中间那套把“原始文本”翻译成“可计算叙事因子”的发动机。

它一旦做得好，系统看到的是主线、催化、板块、龙头和资金共振。

它一旦做得差，系统看到的就只是噪音、误判和错误归因。

所以，和 ChatGPT 深化词典质量，不是一个“可做可不做”的优化项，而是整个叙事雷达、D 组共振、以及未来 C/D 两组叙事协同选股能力的基础工程。
